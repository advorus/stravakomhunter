# Agent Instructions

- After completing a singular task, commit the relevant changes, open a pull request, and review that pull request before considering the task complete.
- If the user asks to "approve" a pull request, treat that as: perform a full review of the changed code and then merge the pull request if appropriate. Do not require a separate reviewer for this repository because the user is the sole contributor.
- Keep commits focused on the task at hand and avoid staging unrelated local changes.
- Run the relevant verification command before opening the pull request whenever the project provides one.
